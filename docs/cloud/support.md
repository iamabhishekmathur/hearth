# Cloud Support

Hearth Cloud support helps workspace admins investigate hosted-workspace issues, account questions, and operational problems that are not caused by local self-hosted infrastructure.

[[toc]]

## Before Contacting Support

Collect the smallest useful reproduction:

- Workspace name.
- User email or role affected.
- Feature area: chat, tasks, routines, integrations, memory, admin, billing, or SSO.
- Approximate time of the issue.
- Screenshots or error messages if available.
- Whether the issue affects one user, one team, or the whole workspace.
- Recent admin changes, integration changes, or SSO changes.

## Self-Service Checks

Before escalating:

1. Confirm the user has the expected role and team assignment.
2. Re-test failing integrations in **Settings > Integrations**.
3. Check governance violations if a message or action was blocked.
4. Check compliance settings if sensitive content was scrubbed.
5. Check audit logs for recent admin changes.
6. Try a new chat session if an old session has stale context.
7. Review routine run history if an automation failed.

## Severity Guide

| Severity | Examples |
|---|---|
| Critical | Workspace-wide outage, broad login failure, data access incident, or security concern. |
| High | Major feature unavailable for a team, repeated routine failures, broken SSO rollout, or critical integration failure. |
| Normal | Single-user issue, configuration question, unexpected model behavior, non-urgent integration issue. |
| Low | Documentation question, feature clarification, or minor UI issue. |

Use your current support agreement for exact channels and response targets.

## What Support Can Usually Help With

- Workspace access and admin configuration.
- Integration setup and connector failures.
- SSO setup and callback issues.
- Governance and compliance configuration questions.
- Unexpected routine, task, or chat behavior.
- Billing or plan questions.
- Data export and workspace offboarding process.
