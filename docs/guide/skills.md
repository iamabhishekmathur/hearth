# Skills

Applies to: Hearth Cloud and self-hosted Hearth.

Skills are reusable AI workflows. They let one person's successful pattern become available to the whole team.

[[toc]]

## What Skills Do

Skills package instructions, examples, and supporting files so Hearth can perform a specialized workflow more consistently.

Use skills for:

- Code review styles.
- Research workflows.
- Launch checklists.
- Support triage.
- Writing formats.
- Migration safety.
- Team-specific operating procedures.

## Browsing and Installing

The Skills page lets users browse available skills, inspect details, install or uninstall them, and use installed skills in chat and tasks.

## Creating Skills

Users can create custom skills from patterns that work well. A skill should describe when to use it, the process to follow, and any examples or templates needed for consistent output.

Developers can read the [SKILL.md Format](/developers/skill-format) for the file-level specification.

## Review and Governance

Organizations can review skill proposals before making them broadly available. Admins can govern which skills are trusted and how they are distributed.

See [Skill Governance](/admin/skill-governance) and [Governance](/admin/governance) for policy controls.

## Recommendations

Hearth can recommend skills based on user activity and completed work. This helps workflows spread from power users to the rest of the team.
