# Product Guide

The product guide covers the shared Hearth experience across Hearth Cloud and self-hosted Hearth. Edition-specific setup, operations, billing, and infrastructure details live in the [Cloud](/cloud/) and [Self-Hosting](/self-hosting/) sections.

[[toc]]

## Core Workflow

Hearth gives teams a shared workspace for AI-assisted work:

1. Use [Chat](/guide/chat) to collaborate with the AI and teammates.
2. Generate [Artifacts](/guide/artifacts) when the answer should become a document, code block, diagram, table, or HTML preview.
3. Promote work into [Tasks](/guide/tasks) when the agent should plan, execute, and return for review.
4. Save repeated work as [Routines](/guide/routines).
5. Let [Memory](/guide/memory), [Skills](/guide/skills), [Activity](/guide/activity), and the [Decision Graph](/guide/decisions) compound the team's knowledge.

## Feature Map

| Feature | What it is | Start here |
|---|---|---|
| Chat | Shared AI sessions with streaming, tools, presence, reactions, sharing, and task promotion. | [Chat](/guide/chat) |
| Artifacts | Structured outputs that live beside chat and can be revised or promoted to tasks. | [Artifacts](/guide/artifacts) |
| Tasks | Kanban workflow for delegated work, planning, execution, review, and replanning. | [Tasks](/guide/tasks) |
| Memory | Org, team, user, and session context with semantic recall. | [Memory](/guide/memory) |
| Routines | Scheduled and event-driven AI workflows with state, templates, gates, and delivery. | [Routines](/guide/routines) |
| Skills | Reusable workflows that can be installed, created, reviewed, and recommended. | [Skills](/guide/skills) |
| Activity | Cross-workspace event stream, reactions, digests, and proactive signals. | [Activity Feed](/guide/activity) |
| Decisions | Organizational decision intelligence, outcomes, patterns, and principles. | [Decision Graph](/guide/decisions) |

## Edition Notes

The product pages apply to both editions unless a section calls out a difference:

- **Hearth Cloud:** Hearth operates the infrastructure. Admins configure the workspace.
- **Self-hosted OSS:** Your team operates the infrastructure. Admins configure the workspace and operators configure deployment, storage, secrets, backups, upgrades, and monitoring.

For deployment responsibilities, see [Cloud vs Self-Hosted](/getting-started/comparison).
